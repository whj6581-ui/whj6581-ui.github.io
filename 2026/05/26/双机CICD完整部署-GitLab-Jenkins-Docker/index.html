双机CI/CD完整部署-GitLab+Jenkins+Docker

本文记录了在两台虚拟机上从零搭建完整 CI/CD 流水线的全过程，实现代码推送后自动构建、部署应用。

架构说明

  • 虚拟机A（GitLab）：192.168.2.108，负责代码托管
  • 虚拟机B（Jenkins）：192.168.2.104，负责自动构建和部署
  • 应用栈：Flask + MySQL + Nginx，全部跑在 Docker 容器里

整体流程：开发者推送代码 → GitLab 触发 Webhook → Jenkins 自动拉取代码 → 构建镜像 → 部署容器 → 健康检查

一、两台机器公共初始化

两台虚拟机都要执行：

二、虚拟机A — 安装 Docker 并启动 GitLab

GitLab 启动后访问 http://192.168.2.108，用 root 登录并修改密码，然后创建项目 myapp。

还需要开启内网 Webhook 支持：
进入 http://192.168.2.108/admin/application_settings/network → Outbound requests → 勾选 Allow requests to the local network → Save

三、虚拟机B — 安装 Jenkins

安装 Jenkins 插件

安装 Docker 和 Docker Compose

四、配置 SSH 密钥互信

GitLab 添加公钥：右上角头像 → Edit Profile → SSH Keys → 粘贴公钥

Jenkins 添加私钥：Manage Jenkins → Credentials → Add Credentials → 类型选 SSH Username with private key → ID 填 gitlab-ssh

五、应用代码结构

项目包含以下文件：

  • app.py：Flask 应用，提供 / 和 /health 接口
  • requirements.txt：Python 依赖
  • Dockerfile：构建应用镜像
  • docker-compose.yml：编排 app + mysql + nginx 三个容器
  • nginx/nginx.conf：反向代理配置
  • Jenkinsfile：流水线定义，包含拉取代码、构建镜像、停止旧容器、启动新容器、健康检查五个阶段

推送代码到 GitLab：

六、Jenkins 配置流水线

  1. New Item → 名称 myapp-pipeline → Pipeline → OK
  2. Build Triggers → 勾选 Build when a change is pushed to GitLab，生成 Secret token 并保存
  3. Pipeline → Pipeline script from SCM → Git → 填入仓库地址，选 gitlab-ssh 凭据，分支 */main
  4. Save

七、GitLab 配置 Webhook

项目 → Settings → Webhooks → Add new webhook

  • URL：http://192.168.2.104:8088/project/myapp-pipeline
  • Secret token：填 Jenkins 生成的 token
  • Trigger：勾选 Push events
  • SSL verification：取消勾选

配置完点 Test → Push events，Jenkins 收到触发说明打通成功。

八、验证端到端流程

推送后去 Jenkins 查看是否自动触发构建，流水线全部绿色即部署成功。

九、MySQL 自动备份

通过 crontab 每天凌晨 2 点自动执行备份，同时每 5 分钟检查一次服务健康状态。

踩过的坑

  • GitLab 容器启动较慢，docker ps 显示 healthy 才算真正可用，一般需要等 3-5 分钟
  • Jenkins 插件从官方源下载很慢，需要手动配置 DNS 解析指向国内节点
  • GitLab 默认禁止内网 Webhook，需要在管理后台手动开启否则流水线无法触发
  • Docker Compose 新版命令是 docker compose（无横线），旧版是 docker-compose，Jenkinsfile 里两个都写上做兼容
