从零搭建 Kubernetes 三节点集群实战记录

发表于2026-06-01|更新于2026-06-01|云原生
|浏览量:

背景

本文记录了在本地虚拟机环境下，从零搭建一个 Kubernetes 三节点集群的完整过程，包括遇到的坑和解决方案，适合初学者参考。

环境规划

主机名 IP 角色 配置
master 192.168.2.106 Control Plane 2核2G
worker1 192.168.2.109 Worker 2核2G
worker2 192.168.2.110 Worker 2核2G
  • 操作系统：CentOS 7
  • Kubernetes 版本：1.28.2
  • 容器运行时：containerd 1.6.33
  • 网络插件：Flannel v0.25.1

主要步骤

1. 系统初始化

三台机器都需要完成以下配置：

  • 关闭 SELinux 和防火墙（K8s 自行管理网络规则）
  • 关闭 swap（K8s 要求强制关闭，否则 kubelet 无法启动）
  • 加载 overlay 和 br_netfilter 内核模块
  • 配置内核参数开启 IP 转发和网桥 iptables 规则
  • 配置 /etc/hosts 实现节点间主机名解析

2. 安装 containerd

K8s 1.24 版本后移除了对 Docker 的直接支持，改用 containerd 作为容器运行时。

关键配置：

  • 设置 SystemdCgroup = true（与 kubelet 的 cgroup 驱动保持一致，否则节点会频繁重启）
  • 将 sandbox_image 替换为阿里云镜像地址，解决 registry.k8s.io/pause 在国内无法拉取的问题

3. 安装 kubeadm/kubelet/kubectl

使用阿里云的 yum 源安装 1.28.2 版本，锁定版本避免自动升级导致集群不稳定。

4. Master 初始化

通过 --image-repository 指定阿里云镜像仓库，解决 k8s.io 相关镜像在国内无法拉取的问题。

5. Worker 节点加入

复制 kubeadm init 输出的 join 命令在两台 Worker 上执行即可。注意 token 默认 24 小时过期，过期后需重新生成。

6. 安装 Flannel 网络插件

K8s 集群默认没有网络插件，Pod 之间无法通信，需要手动安装。

选择 Flannel 的原因：配置简单，适合学习环境，使用 vxlan 模式实现跨节点网络覆盖。

由于 docker.io/flannel 镜像国内无法直接拉取，使用华为云镜像源替代后手动打 tag：

注意： 三台机器都要提前拉取 Flannel 镜像，否则 Worker 节点上的 Flannel Pod 会 ImagePullBackOff。

7. 部署 Dashboard

通过 NodePort 30443 对外暴露 Dashboard，使用 HTTPS 访问。创建 ClusterAdmin 权限的 ServiceAccount，通过 Token 方式登录。

踩坑记录

坑1：kubeadm init 超时

报错 [kubelet-check] Initial timeout of 40s passed，原因是 containerd 配置里的 pause 镜像地址是 registry.k8s.io/pause，国内访问不到。

解决方法：修改 /etc/containerd/config.toml，将 sandbox_image 改为 registry.aliyuncs.com/google_containers/pause:3.9，重启 containerd 后重新 init。

坑2：Flannel Pod ImagePullBackOff

只在 Master 上拉了 Flannel 镜像，忘了 Worker 节点也需要，导致 Worker 上的 Flannel Pod 一直拉取失败。

解决方法：在三台机器上都执行镜像拉取和打 tag 操作。

坑3：DNS 解析失败导致 yum 安装失败

resolv.conf 被 NetworkManager 动态覆盖，DNS 配置不稳定，导致 yum 源间歇性解析失败。

解决方法：在 /etc/NetworkManager/NetworkManager.conf 中添加 dns=none，禁止 NetworkManager 覆盖 resolv.conf。

最终结果

三台节点全部 Ready，Flannel 网络插件正常，Dashboard 可视化界面可访问，Pod 调度测试通过（3 个 nginx Pod 均匀分布在两个 Worker 节点上）。

总结

搭建过程中最大的难点是国内网络环境问题，几乎每个组件的镜像都需要找替代源。整理出来的规律是：

  • K8s 核心组件镜像 → 阿里云（registry.aliyuncs.com/google_containers）
  • Docker Hub 镜像（flannel、nginx 等）→ 华为云（swr.cn-north-4.myhuaweicloud.com/ddn-k8s/docker.io/）
  • pip 包 → 清华源（https://pypi.tuna.tsinghua.edu.cn/simple）
  • yum 包 → 阿里云（mirrors.aliyun.com）
