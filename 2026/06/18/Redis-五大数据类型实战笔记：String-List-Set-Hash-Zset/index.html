二、List（列表）

Redis 的 List 本质是一个双向链表，支持从两端插入和弹出，既可以当栈用，也可以当队列用。

查询 lrange lindex llen
删除 lpop rpop lrem ltrim
修改 lset

实现 命令组合
栈（先进后出） lpush + lpop
队列（先进先出） lpush + rpop

List 在实际场景中最常用于实现简单的消息队列，比如生产者用 lpush 写入任务，消费者用 rpop 取出处理。

Set 是无序、不重复的字符串集合，天然支持去重，并且提供交集、并集、差集运算。

四、Hash（哈希）

Hash 相当于”在 key 里再放一个 map”，最适合存储对象类型的数据，因为可以单独读写某个字段，不需要像 JSON 字符串那样整体反序列化。

Hash vs JSON字符串存对象

方式 优点 缺点
String 存 JSON 简单直观 改一个字段要整体重写，浪费带宽
Hash 可单独修改某个字段，节省内存和带宽 嵌套对象（对象里还有对象）处理起来比较麻烦

存储用户信息、商品信息这类结构化对象时，Hash 通常是比 JSON 字符串更优的选择。

五、Zset（有序集合）

Zset = Set + 分数（score），元素本身唯一不重复，但会按分数自动排序，是实现排行榜的标准方案。

热搜榜/热度榜 score=点击量，zincrby 实时累加
延迟队列 score=执行时间戳，定时扫描取出到期任务

小结

五种数据类型的选型可以简单总结为：

类型 核心特点 典型场景
String 简单key-value，支持原子自增 计数器、缓存、分布式锁
List 双向链表，有序可重复 消息队列、最新列表、栈
Set 无序不重复，支持集合运算 标签去重、共同好友、抽奖
Hash key中嵌套字段，可单独读写 存储对象（用户信息、商品信息）
Zset 有序不重复，按分数排序 排行榜、热度榜、延迟队列
